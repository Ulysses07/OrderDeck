1. PhoneEntryDialog'da `abc` gir → **Kaydet ve Aç**
2. Inline kırmızı uyarı: "Geçersiz telefon numarası. 10 haneli TR mobil numara girin."

### 4b. Geçersiz telefon (Phase 4f form)

2. **Tamamla** → Razor validation: "Geçersiz telefon numarası. 10 haneli TR mobil numara girin."
3. Form yeniden render edilir, submission kaydedilmez.

### 4c. WhatsApp Desktop kurulu değil
1. WhatsApp Desktop kurulu olmayan makinede 📱 tıkla
2. OS default handler `wa.me` linkini açar → web.whatsapp.com fallback (giriş gerekirse istemden sonra mesaj görünür).

### 4d. Settings template boş bırakılırsa
1. Settings → Mesaj Şablonu alanını sil → Kaydet
2. Yeni template **boş string** olarak persist edilir
3. WhatsApp açıldığında mesaj boş gönderilir (broadcaster manuel yazabilir)
4. **Note:** PaymentRequestService template fallback'ı YOK — boş template = boş mesaj. Eğer fallback istenirse Phase 4h.

### 4e. No ended session, filter aktif
1. Henüz `EndedAt` dolu session yokken CustomerSearch'te filter işaretle
2. Grid boş kalır (CustomerService.GetLastStreamShoppers empty döner)
3. UI'da specific InfoBar yok — grid boşluğu yeterli (YAGNI)

## 5. Telefon format kabulleri

`PhoneNormalizer.NormalizeTr` aşağıdakileri kabul eder, hepsi `+905551234567` çıkarır:
- `5551234567` (10 hane)
- `05551234567` (11 hane, leading 0)
- `905551234567` (12 hane, no plus)
- `+905551234567` (zaten E.164)
- `+90 555 123 45 67` (boşluklu)
- `0 555 123-45-67` (boşluk + dash)
- `(0555) 123 45 67` (parantez)

Reddedilenler (null döner):
- Boş/null/whitespace
- 10 haneden az/14 haneden fazla
- `abc`, alpha karakterler
- `+12025551234` (yurt dışı, sadece TR)
